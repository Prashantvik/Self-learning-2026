## 🎯 Company-Specific Resources

### Google
#### Interview Process
- **Guide**: [Google Technical Interview Guide](https://techdevguide.withgoogle.com/)
- **Focus**: System design, scalability, optimization
- **Coding**: Python, Java preferred
- **Rounds**: 4-5 technical rounds + behavioral

#### Specific Preparation
- **LeetCode**: [Google Tagged Questions](https://leetcode.com/company/google/)
- **Count**: 500+ problems (solve top 100)
- **Focus Areas**: Trees, Graphs, DP, System Design
- **Data Engineering**: Focus on data processing, pipelines

#### Sample Questions
1. Design a data pipeline for processing user events
2. Implement a distributed hash table
3. Design YouTube's recommendation system
4. Optimize a slow-running ETL job

### Uber
#### Interview Process
- **Focus**: Real-world problems, optimization, scalability
- **Coding**: Python/Java/Scala
- **Rounds**: 3-4 technical + behavioral
- **Unique**: Often includes take-home assignments

#### Specific Preparation
- **LeetCode**: [Uber Tagged Questions](https://leetcode.com/company/uber/)
- **Count**: 200+ problems (solve top 70)
- **Focus Areas**: Graphs (shortest path), DP, Arrays
- **Data Engineering**: Real-time processing, Kafka, Spark

#### Sample Questions
1. Design Uber's ETAs calculation system
2. Process real-time ride requests efficiently
3. Design a surge pricing algorithm
4. Implement dynamic graph algorithms

### Adobe
#### Interview Process
- **Focus**: Algorithms, data structures, creative problem-solving
- **Coding**: Python/Java/C++
- **Rounds**: 3-4 technical + behavioral
- **Unique**: Sometimes includes domain-specific questions

#### Specific Preparation
- **LeetCode**: [Adobe Tagged Questions](https://leetcode.com/company/adobe/)
- **Count**: 150+ problems (solve top 50)
- **Focus Areas**: Strings, DP, Arrays, Design Patterns
- **Data Engineering**: Analytics, reporting, data transformation
