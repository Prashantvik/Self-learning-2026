## 🔄 Problem-Solving Methodology

### Step 1: Understanding (5 minutes)
```
□ Read problem statement 2-3 times
□ Identify input/output format
□ Understand constraints
□ Think of edge cases
□ Ask clarifying questions (in interviews)
```

### Step 2: Approach (10-15 minutes)
```
□ Think of brute force solution first
□ Identify the pattern/algorithm needed
□ Think of optimizations
□ Estimate time/space complexity
□ Choose the best approach
```

### Step 3: Implementation (15-30 minutes)
```
□ Write clean, readable code
□ Handle edge cases
□ Use meaningful variable names
□ Add comments for complex logic
□ Test with sample inputs
```

### Step 4: Testing (5-10 minutes)
```
□ Test with given examples
□ Test edge cases (empty input, single element, etc.)
□ Test large inputs (if applicable)
□ Verify time complexity is acceptable
```

### Step 5: Optimization (5-10 minutes)
```
□ Can time complexity be improved?
□ Can space complexity be reduced?
□ Are there any redundant operations?
□ Can the code be cleaner?
```

### Step 6: Documentation (5 minutes)
```
□ Add problem to tracking sheet
□ Note the pattern used
□ Record time taken
□ Note any difficulties faced
□ Add to review list if needed
```

## 🔥 Emergency Protocols

### When Stuck on a Problem (30+ minutes)
1. **Take a Break**: 10-15 minutes away from screen
2. **Simplify**: Try smaller test cases
3. **Pattern Check**: Is this a known pattern?
4. **Hint Peek**: Look at one hint (not full solution)
5. **Time Box**: Set 15 more minutes, then move to solution
6. **Learn**: Understand solution completely
7. **Retry**: Solve again after 2-3 days
